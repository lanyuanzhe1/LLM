## 三、完整操作流程（6 步）

### 登录

```
浏览器打开：https://training.xfyun.cn/modelSquare
右上角登录 → 微信扫码 / 手机号
```

### 第 1 步：准备微调数据集

格式要求：
- 文件格式：JSON / JSONL / CSV
- 数据格式：**Alpaca**（instruction-input-output）或 ShareGPT（多轮对话）
- 数据量：≥ 2000 条推荐，< 500 条效果不明显

```json
[
  {
    "instruction": "你是一位粮食储藏专家...",
    "input": "储粮害虫有哪些防治方法？",
    "output": "储粮害虫防治...（专业回答）"
  }
]
```

平台提供辅助工具：
- **问答对抽取**：从 TXT 文本自动切分 QA 对
- **数据增强**：1 条种子数据 → 泛化 1~5 倍
- **Prompt 模板**：50+ 预设模板

### 第 2 步：选择基础模型 + 创建精调任务

```
模型广场 → 浏览模型 → 选择目标模型 → 点击「精调此模型」
填写：模型名称、描述
```

### 第 3 步：上传数据集 + 配置参数

上传准备好的 JSONL 文件，配置 LoRA 参数：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| **精调方法** | LoRA | < 1% 参数更新，几分钟完成 |
| **学习率** | 5e-5 | Qwen 系列推荐值 |
| **训练轮数 (Epochs)** | 4-5 | 2000+ 条数据 |
| **LoRA Rank** | 16 | 7B 模型推荐 |
| **LoRA Alpha** | 32 | = Rank × 2 |
| **最大序列长度** | 2048 | 按最长样本调整 |
| **Dropout** | 0.05 | 防过拟合 |

### 第 4 步：提交训练 + 监控

```
点击「提交」→ 训练开始 → 观察 Loss 曲线
- Loss 持续下降收敛 ✅
- 训练 Loss↓ 但验证 Loss↑ → 过拟合，减少 Epochs
- Loss 不收敛 → 调低学习率
```

LoRA 微调通常 **几分钟到十几分钟** 完成。

### 第 5 步：评估

平台提供两种评估方式：
- **相似度打分**：对比微调前后输出
- **裁判员打分**：LLM 自动评估效果
- **手动对比**：在线体验中心直接对比

### 第 6 步：发布为 API 服务

```
训练完成 → 点击「发布为服务」
  → 创建应用（名称 + 分类 + 功能描述）
  → 绑定模型版本
  → 获得：
      - API Key
      - Base URL（如 maas-api.cn-huabei-1.xf-yun.com）
      - serviceId
```

调用示例（兼容 OpenAI SDK）：
```python
from openai import OpenAI

client = OpenAI(
    api_key="你的API_Key",
    base_url="https://maas-api.cn-huabei-1.xf-yun.com/v1",
)

response = client.chat.completions.create(
    model="你的serviceId",   # MaaS 平台分配的服务编号
    messages=[{"role": "user", "content": "储粮害虫怎么防治？"}],
)
```

## 四、模型推荐

### 你的场景：粮食储藏垂类微调

| 推荐 | 模型 | 理由 |
|------|------|------|
| ⭐ **首选** | **Qwen-2.5-7B-Instruct** | 7B 参数，中文能力顶级，LoRA 微调成本最低，RTX 4050 可本地推理 |
| 备选 | Spark-Lite | 讯飞平台原生，可能有免费额度，但微调灵活性不如 Qwen |
| 高分方案 | Qwen-2.5-14B | 精度更高，但微调耗时更长，推理成本更高 |

### 为什么选 Qwen-2.5-7B 而不是 DeepSeek？

- DeepSeek 在 MaaS 上主要提供**推理 API**，精调授权可能受限
- Qwen 系列是 MaaS 平台**主力支持的微调模型**，教程和经验最丰富
- 7B 规模正好：微调快、推理快、精度满足垂类需求
- 开源社区活跃，微调参数配置（LoRA target_modules 等）有成熟方案

> `★ Insight ─────────────────────────────────────`
> **为什么不推荐直接用 DeepSeek-V3（685B MoE）微调？** MoE（混合专家）架构虽然参数总量大，但每次推理只激活一部分，实际计算量 ≈ 37B。不过微调 MoE 模型比微调 Dense 模型（如 Qwen-7B）复杂得多——需要特殊处理路由专家、负载均衡，而且 685B 权重即使是 LoRA 也需要大量 GPU 显存。除非你有极端精度需求 + 大量 GPU 预算，否则 7B Dense 模型微调是 90% 垂类场景的最优解。
> `─────────────────────────────────────────────────`

## 五、完整 RAG + 微调架构

```
knowledge/ 文档
    │
    ├──→ [已上线] 向量库 vector_store/ → sklearn 检索
    │
    └──→ [下一步] 构造微调数据集
              │
              ├── 从文档抽 QA 对（chatdoc qa/extract 或讯飞数据工具）
              ├── 数据增强（泛化到 2000+ 条）
              │
              ▼
         MaaS 平台 → Qwen-2.5-7B LoRA 微调
              │
              ▼
         部署为 API（获得 model serviceId）
              │
              ▼
         RAG：检索（本地向量库） + 生成（微调模型 API）
              │
              ▼
         评估 → 迭代 → 竞赛提交
```
