# 星辰工作流联调指南

本指南只描述本地准备、受限工具暴露和在线联调。离线测试不访问网络；
`tests/online` 默认只收集并跳过，只有显式设置 `RUN_ONLINE=1` 后才会使用已配置的
真实云服务。向量重建同样需要有权限的操作员主动执行。

## 1. macOS/Linux：安装与配置

在仓库根目录启用项目环境，再安装开发依赖：

```bash
conda activate LLM
python -m pip install -r requirements-dev.txt
cp .env.example .env
```

编辑本地 `.env`，填写所需讯飞配置和独立的 `TOOLS_SERVICE_TOKEN`；不要提交该文件
或在日志、提示词中显示其值。`build_vector_store.py` 从进程环境读取 Embedding 配置，
所以重建前加载 `.env`：

```bash
set -a
source .env
set +a
```

`.env.example` 中的 MaaS 和星辰 `*_TIMEOUT_SECONDS` 是覆盖连接、发送和完整流式
读取的单次总时限；持续到达小帧不会延长时限。`*_MAX_FRAMES`、
`*_MAX_PAYLOAD_BYTES`、`*_MAX_ANSWER_CHARS` 分别限制每次调用的云端帧数、累计
响应字节和回答字符数。`GATEWAY_MAX_BUFFER_CHARS` 是公开网关在引用安全校验前的
独立回答缓冲上限。保留示例默认值即可作为安全起点；所有限制必须是正整数，按实际
容量调整时应先离线验证。超时或超限会结束并清理当前流，返回稳定错误，未验证回答
不会输出。

## 2. macOS/Linux：离线验证、向量重建与启动

离线验证不应触发任何云端请求：

```bash
python -m pytest -m "not online" -q
```

在未设置 `RUN_ONLINE` 的 shell 中，以下在线目录命令会全部收集并跳过，
不会调用云服务：

```bash
python -m pytest tests/online -v
```

云适配器 smoke 不需要本地 FastAPI 或 `LOCAL_PUBLIC_API_URL`，但需要完整、有效的
Embedding、MaaS 和工作流凭据；其中工作流测试还需要已发布的 Flow ID。满足这些
provider 前置条件后，才显式运行仅包含云适配器的文件：

```bash
RUN_ONLINE=1 python -m pytest tests/online/test_cloud_services.py -v
```

以下是无交互的向量重建命令；它跳过构建后的交互式搜索，但会调用已配置的
Embedding 服务：

```bash
SKIP_VECTOR_SEARCH=1 python build_vector_store.py
```

完成后必须存在 `vector_store/vectors.npy` 和
`vector_store/chunks_metadata.json`。只有凭据、有效 vector store、已发布的 Flow ID 和
受限公网 HTTPS 工具 URL 都齐全时才运行端到端测试。服务前台启动会阻塞，因此使用两个
终端：

终端 A 启动单 worker 服务并保持运行：

```bash
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 1
```

终端 B 使用同一个已配置环境运行本地 E2E；命令前缀只作用于本次进程，不污染 shell：

```bash
RUN_ONLINE=1 LOCAL_PUBLIC_API_URL=http://127.0.0.1:8000 python -m pytest tests/online/test_end_to_end.py -v
```

`GET /health` 是存活探针，始终以 `200 {"status":"ok"}` 表示进程可响应，
不验证向量或云配置。`GET /ready` 是就绪探针：仅当向量库和静态云配置都有效时
返回 `200` 及向量数量、维度；在两者都有效前保持 `503`（缺向量为
`VECTOR_STORE_NOT_READY`，云配置无效为 `CLOUD_CONFIG_NOT_READY`）。该检查不发起网络请求。

## 3. Windows PowerShell：配置、验证、重建与启动

在仓库根目录执行与上面等价的操作：

```powershell
conda activate LLM
python -m pip install -r requirements-dev.txt
Copy-Item .env.example .env
# 编辑 .env 后，将 KEY=VALUE 行载入当前进程；不要把 .env 提交到仓库。
Get-Content .env | Where-Object { $_ -and -not $_.StartsWith('#') } | ForEach-Object {
  $name, $value = $_.Split('=', 2)
  [Environment]::SetEnvironmentVariable($name, $value, 'Process')
}
python -m pytest -m 'not online' -q
# 未设置 RUN_ONLINE 时全部收集并跳过，不调用云服务。
python -m pytest tests/online -v
# 云适配器 smoke 需要有效 Embedding、MaaS 和工作流凭据；工作流还需要已发布 Flow ID。
$env:RUN_ONLINE = "1"
python -m pytest tests/online/test_cloud_services.py -v
Remove-Item Env:RUN_ONLINE -ErrorAction SilentlyContinue
$env:SKIP_VECTOR_SEARCH = "1"
python build_vector_store.py
```

PowerShell 的向量重建同样不交互，但需要有效 Embedding 额度；运行后可用
`GET /health` 和 `GET /ready` 按上一节的语义检查服务。只有凭据、有效 vector store、
已发布的 Flow ID 和受限公网 HTTPS 工具 URL 都齐全时才运行 E2E；前台服务必须保持运行。

终端 A：

```powershell
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 1
```

终端 B：

```powershell
$env:RUN_ONLINE = "1"
$env:LOCAL_PUBLIC_API_URL = "http://127.0.0.1:8000"
python -m pytest tests/online/test_end_to_end.py -v
Remove-Item Env:LOCAL_PUBLIC_API_URL -ErrorAction SilentlyContinue
Remove-Item Env:RUN_ONLINE -ErrorAction SilentlyContinue
```

## 4. 暴露工具接口

调用方向固定为：面向用户的本地或另行授权的公开 `/v1/*` 调用星辰工作流，星辰只允许调用经过认证的 `/tools/v1/*`，`/tools/v1/*` 绝不调用星辰工作流，避免形成递归调用。这里的 `/v1/*` 工作流入口具体是 `/v1/chat` 和 `/v1/cases/analyze`；`/v1/sources/{evidence_id}` 是本地证据查询，不发起工作流。

使用团队批准的 HTTPS 隧道或反向代理把本机 8000 端口映射到公网，并配置路径白名单，仅允许 `/tools/v1/*`。同一公网入口上的 `/v1/*`、`/health`、`/ready`、`/openapi.json`、文档和管理路径均保持不暴露，除非另行授权并配置相应访问控制。只把这个受限公网地址配置给星辰工具；不要把隧道访问令牌写入仓库。

这是隧道或反向代理的暴露策略，不是删除或停用应用路由；这些路由仍可按本地或其他已授权入口的用途保留。

## 5. 创建并发布工作流

按 `workflow/README.md` 和 `workflow/tool_contracts.json` 创建四个工具节点、嵌套
JSON 映射及异常分支。仓库提供的是精确工具 schema，不是已发布的星辰 Flow export；
完成平台调试并发布为 API 后，才把获得的 Flow ID 写入本地 `.env` 的
`XF_WORKFLOW_FLOW_ID`。

## 6. Trace 定位

在星辰发布管理中打开工作流详情和 Trace 日志，用本地日志中的 `REQUEST_ID` 对照工作流输入参数，定位失败节点。日志中不得记录任何密钥。
